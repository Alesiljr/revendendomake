## Description

Este story implementa o design system completo do Revendendo Make com base no documento `docs/front-end-spec.md`: tokens de design como CSS variables em `globals.css`, configuração do Tailwind com a paleta framboesa/dourado do projeto, fontes Playfair Display e Inter, e todos os componentes de layout global (Header, Footer, MobileDrawer, WhatsApp FAB).

O resultado é uma biblioteca de componentes reutilizável que todos os stories dos Epics 2, 3 e 4 irão consumir. Os componentes devem ser acessíveis (WCAG 2.1 AA), responsivos (mobile-first), e construídos sobre shadcn/ui customizado com os tokens do design system.

Este story também cria os componentes base do layout admin (AdminSidebar, AdminHeader) e os utilitários de formatação que serão amplamente usados no projeto (formatPrice, formatDate, buildWhatsAppLink, generateSlug).

### Padrões de Componente Obrigatórios

Todos os componentes seguem o padrão de `docs/coding-standards.md`:

```typescript
// Server Component (padrão — sem diretiva)
interface Props {
  className?: string
}

export function ComponentName({ className }: Props) {
  return <div className={cn('base-classes', className)}>...</div>
}

// Client Component (apenas quando necessário)
'use client'
export function ClientComponent() {
  const [state, setState] = useState(false)
  // ...
}
```

- Sem `React.FC` — usar function declaration
- Props sempre tipadas com `interface` ou `type`
- `cn()` para classes condicionais
- Sem inline styles (exceto transforms dinâmicos)

### Cores — Token Primário de Projeto

A cor primária principal é **`#C9184A`** (framboesa — `--color-primary-700`). É esta cor que deve aparecer em CTAs, botões primários, links ativos do admin, e destaques de navegação.

O dourado (`--color-secondary-500: #d4a843`) é usado para badges de destaque, elementos premium, e acentos visuais — nunca como cor principal de interação.

### shadcn/ui — Não Modificar Componentes Base Automaticamente

Após executar `npx shadcn@latest add`, os arquivos gerados em `src/components/ui/` podem ser customizados localmente. **Nunca re-executar** `npx shadcn@latest add` para um componente já customizado, pois sobrescreve as customizações.

### MobileDrawer — Padrão de Comunicação com Header

O Header é Server Component e não pode ter estado. O `MobileDrawer` é Client Component. A solução é criar um wrapper client `HeaderClient.tsx` que contém o botão hamburger e o drawer, enquanto o `Header.tsx` server component renderiza as partes estáticas (logo, nav links).

### Admin Layout — Auth Server-Side

O `src/app/admin/layout.tsx` deve verificar a sessão server-side para evitar flash de conteúdo protegido:

```typescript
// src/app/admin/layout.tsx
import { redirect } from 'next/navigation'
import { createClient } from '@/lib/supabase/server'

export default async function AdminLayout({ children }: { children: React.ReactNode }) {
  const supabase = await createClient()
  const { data: { user } } = await supabase.auth.getUser()
  
  if (!user) redirect('/login')
  
  return (
    <div className="flex h-screen overflow-hidden">
      <AdminSidebar />
      <div className="flex flex-1 flex-col overflow-hidden">
        <AdminHeader />
        <main className="flex-1 overflow-y-auto p-6">{children}</main>
      </div>
    </div>
  )
}
```
