## Description

Este story cobre toda a infraestrutura externa necessária antes que qualquer linha de código da aplicação possa ser escrita. O objetivo é garantir que o repositório GitHub exista, o domínio `revendendomake.com.br` esteja registrado e apontando para o Vercel, o projeto Supabase esteja provisionado, e o pipeline de deploy contínuo (GitHub → Vercel) esteja ativo.

Este story é **puramente de infraestrutura e configuração** — não envolve código de aplicação Next.js, schema de banco de dados ou design system. Todo o trabalho se passa em dashboards externos (GitHub, Vercel, Registro.br, Supabase, UptimeRobot) e na criação de um único arquivo de template (`.env.local.example`).

Ao final deste story, qualquer desenvolvedor deve conseguir clonar o repositório, copiar `.env.local.example` para `.env.local`, preencher as variáveis reais e ter um ambiente de desenvolvimento funcional apontando para os serviços de produção.

## Technical Notes

- **Registro de domínio .com.br:** requer CPF ou CNPJ brasileiro. Processo no Registro.br pode levar até 3 dias úteis para propagação completa de DNS. Planejar com antecedência.
- **Vercel DNS propagation:** após configurar os registros no Registro.br, pode levar até 48h para o Vercel validar e provisionar o certificado SSL. Este é o passo com maior lead time do story.
- **Variáveis de ambiente no Vercel:** variáveis com prefixo `NEXT_PUBLIC_` são expostas no bundle do cliente — nunca colocar segredos com esse prefixo. `SUPABASE_SERVICE_ROLE_KEY` e `RESEND_API_KEY` nunca devem ter o prefixo `NEXT_PUBLIC_`.
- **Supabase região:** escolher sa-east-1 (São Paulo) para menor latência com usuários brasileiros. Se não disponível, us-east-1.
- **Branch strategy:** para v1, trabalhar diretamente em `main` ou feature branches curtos. Não há necessidade de `develop` separado neste momento.
- **UptimeRobot plano gratuito:** suporta até 50 monitores com intervalo mínimo de 5 minutos — suficiente para este projeto.
