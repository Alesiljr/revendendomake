## Description

Este story inicializa o projeto Next.js 15 com App Router e configura toda a stack tecnológica base que será usada em todos os stories subsequentes. O resultado é um projeto funcionando em `localhost:3000`, passando em lint e typecheck, com deploy automático ativo no Vercel.

O trabalho inclui: criação do projeto Next.js com TypeScript strict, configuração do Tailwind CSS com as fontes do projeto, instalação e configuração de todos os pacotes da stack (Supabase, shadcn/ui, Framer Motion, Zod, Lucide, next-sitemap), criação dos clientes Supabase (browser e server), criação do middleware de proteção de rotas admin, e configuração do `next.config.ts` com remotePatterns e security headers.

Ao final deste story, o projeto deve fazer build de produção com sucesso, não ter erros de TypeScript, e ter deploy automático no Vercel funcionando — mesmo sem nenhuma página real implementada ainda.

  ```typescript
  import { createServerClient } from '@supabase/ssr'
  import { cookies } from 'next/headers'
  import type { Database } from '@/lib/supabase/types'

  export async function createClient() {
    const cookieStore = await cookies()
    return createServerClient<Database>(
      process.env.NEXT_PUBLIC_SUPABASE_URL!,
      process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
      {
        cookies: {
          getAll() { return cookieStore.getAll() },
          setAll(cookiesToSet) {
            try {
              cookiesToSet.forEach(({ name, value, options }) =>
                cookieStore.set(name, value, options)
              )
            } catch { /* middleware handles session refresh */ }
          },
        },
      }
    )
  }
  ```

### Padrões Obrigatórios (conforme `docs/coding-standards.md`)

- Sem `any` em TypeScript — usar `unknown` com type guards
- Imports sempre absolutos com `@/` — nunca relativos com `../`
- `import type` para importações de apenas tipos
- Server Components como padrão — `'use client'` apenas quando necessário
- Inline styles proibidos — usar Tailwind utilities

### Supabase SSR vs Supabase JS

Usar `@supabase/ssr` (não `@supabase/auth-helpers-nextjs` que está deprecated) para compatibilidade total com Next.js 15 App Router.

### Framer Motion e SSR

Componentes Framer Motion que usam hooks de animação devem ser importados com `dynamic()`:
```typescript
const MotionDiv = dynamic(() => import('framer-motion').then(m => m.motion.div), { ssr: false })
```
Isso evita hidrações desnecessárias. Documentar esse padrão nos comentários dos componentes que o usarem.

### shadcn/ui Init

O `shadcn init` vai perguntar sobre:
- Style: "New York" (recomendado para este projeto)
- Base color: "Neutral"
- CSS variables: "Yes"

Os componentes shadcn/ui individuais serão adicionados na Story 1.3.
